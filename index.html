✅ What Worked Well

  • 40 direct references to ethskills.com — clear value in the hackathon
  • Skills like synthesis-hackathon, ethskills, swap-integration, and x402-payments were heavily used
  • Developers appreciated skills that provided complete flows (e.g., Uniswap quote → approval → swap)
  • Security-focused skills like ethskills-security and ethskills-testing were requested

🚀 Opportunities for Growth

  • Discovery problem: Only 4.6% (35 projects) explicitly mentioned ethskills in their stack, despite 40 URL references — developers may not know what's available
  • Skill onboarding: Most helpful skill feedback mentioned specific workflows (e.g., "taught Hermes the full contract API and CLI commands") — skills should include hands-on examples
  • Skill coverage gaps: Top requests included frontend-design (32), ai-coding (15), agent-browser (5) — consider expanding beyond pure blockchain skills
  • Integration depth: Developers valued skills that integrated with their entire stack (GitHub, deployment, testing, frontend) — not just isolated smart contract tools
  • Moltbook adoption: Only 21% blogged about their projects — could ethskills provide a "project showcase" skill to auto-generate write-ups?

🔮 Strategic Recommendations

  1. Skill marketplace/directory: Make it dead simple to browse and install skills — developers want autocomplete, not hunting through docs
  2. End-to-end workflow skills: Package common patterns (ERC-20 deploy → verify → list on DEX) as single skills
  3. Agent-specific skills: Build skills that teach agents how to use OTHER agents (via ERC-8004, x402, etc.)
  4. Quality over quantity: 216 developers needed web-search — focus on making core skills incredibly robust before expanding catalog
  5. Community contribution: Enable developers to publish their own skills — gitlawb is already building this model
